2. WHAT IS MAMA CARE? – THE CORE DEFINITION
Mama Care is an intelligent maternal health platform that uses artificial intelligence, multi-channel communication (App, USSD, Voice), and community health networks to predict pregnancy risks, coordinate emergency response, and provide continuous personalized guidance – from conception to postpartum.

It is designed for low-resource, high-mortality settings (e.g., rural Tanzania, Kenya, Uganda, Malawi) but scalable to any region.

Three core pillars:

Predict – AI identifies early signs of pre-eclampsia, anemia, preterm labor

Prevent – Automated alerts, nutrition plans, and check-ups

Personalize – Each mother gets a digital health record and tailored advice

5. HOW MAMA CARE WORKS (TECHNICAL & OPERATIONAL MODEL)
Step-by-step user journey:
A. Registration (Day 1)
Mother dials USSD short code (e.g., *270#) or installs app

Registers using simple questions (age, last period, previous complications)

System creates digital maternal health record

B. Ongoing Monitoring
Weekly prompts (USSD or voice call) ask:
“Any swelling in hands/face?”
“Any bleeding?”
“Feeling very tired?”

AI compares answers against medical risk models

C. Risk Detection
If AI detects possible pre-eclampsia, it:

Sends SMS alert to mother: “Please go to clinic today”

Notifies her nearest community health worker (Mdore Mdoueles)

Logs alert in clinic dashboard

D. Emergency Coordination (if severe)
Mother says “emergency” or system detects critical symptoms

Automatically:

Finds nearest ambulance or registered driver

Alerts receiving hospital

Sends mother’s digital record to hospital before arrival

E. Postpartum Follow-up
Continues for 6 weeks after birth

Checks for postpartum hemorrhage, depression, infection

6.3 Sample Queries
Get high-risk pregnancies for a CHW:

sql
SELECT m.name, m.phone, p.lmp, rs.score, rs.risk_type
FROM mothers m
JOIN pregnancies p ON m.id = p.mother_id
JOIN risk_scores rs ON p.id = rs.pregnancy_id
WHERE m.chw_id = 'chw-uuid-here'
  AND p.status = 'active'
  AND rs.level IN ('high', 'critical')
  AND rs.calculated_at > NOW() - INTERVAL '7 days'
ORDER BY rs.score DESC;
Get pending alerts for clinic:

sql
SELECT a.id, m.name, a.alert_type, a.severity, a.message, a.created_at
FROM alerts a
JOIN pregnancies p ON a.pregnancy_id = p.id
JOIN mothers m ON p.mother_id = m.id
WHERE m.clinic_id = 'clinic-uuid-here'
  AND a.resolved_at IS NULL
  AND a.created_at > NOW() - INTERVAL '48 hours'
ORDER BY a.created_at DESC;
Calculate CHW performance:

sql
SELECT 
  chw_id,
  COUNT(DISTINCT pregnancy_id) as mothers_assigned,
  COUNT(a.id) as alerts_received,
  SUM(CASE WHEN a.chw_responded THEN 1 ELSE 0 END) as alerts_responded,
  ROUND(100.0 * SUM(CASE WHEN a.chw_responded THEN 1 ELSE 0 END) / COUNT(a.id), 1) as response_rate
FROM mothers m
LEFT JOIN alerts a ON m.id = a.pregnancy_id
WHERE a.created_at > NOW() - INTERVAL '30 days'
GROUP BY chw_id;

@app.post("/voice/incoming")
async def voice_incoming(request: Request):
    form_data = await request.form()
    speech_result = form_data.get("SpeechResult")
    caller_number = form_data.get("Caller")
    
    # Send to Rasa for intent detection
    rasa_response = requests.post(
        "http://localhost:5005/webhooks/rest/webhook",
        json={"sender": caller_number, "message": speech_result}
    )

    if intent == "emergency":
        trigger_emergency(caller_number)
        return TwilioResponse.say("Dharura. Gari la wagonjwa linakuja.")
    else:
        return TwilioResponse.say(get_response_for_intent(intent))
